Project Type:
AI-assisted Student Operating System

======================================================================

1. PRODUCT VISION

Student Buddy is an integrated Student Operating System that combines academic management and optional personal finance management into a single ecosystem.

The primary objective is NOT to add as many features as possible.

The primary objective is:

"Reduce the number of repetitive micro-decisions a student makes every day."

The system will use:

1. WhatsApp Chatbot
2. Mobile Dashboard App

The app is NOT WhatsApp-first.

It is:

"WhatsApp Assisted + Dashboard Driven Student Operating System"

Users should be able to complete quick actions inside WhatsApp while using the app for visualization, analytics, review and configuration.

The system is designed for a SINGLE USER and NOT for classrooms, CRs or collaborative usage.

======================================================================

2. CORE PRODUCT PHILOSOPHY

The system should reduce daily friction.

Students repeatedly:

- Switch between multiple apps
- Search for information
- Ask friends for details
- Search through WhatsApp groups
- Track attendance manually
- Track expenses manually
- Remember assignment deadlines
- Remember academic events

Student Buddy should automate these repetitive tasks.

AI should be used only when it genuinely adds value.

The app should prioritize simplicity over feature quantity.

======================================================================

Phone Number / WhatsApp Number OTP login

Single user account.

4. MAJOR MODULES

There are two major modules.

4.1 Academic Module (Mandatory)

4.2 Finance Module (Optional)

======================================================================

5. ACADEMIC MODULE

5.1 Timetable Management

Features:

- Weekly repeating timetable
- Subject creation
- Faculty information
- Classroom information
- Timetable visualization
- Timetable notifications
- Timetable OCR import
- Timetable manual editing
- Timetable updates

Users should not upload timetables daily.

Users upload once per semester.

======================================================================

5.2 Attendance Management

Features:

- Before lecture reminders
- After lecture attendance logging
- Present
- Absent
- Cancelled

Attendance calculations:

- Subject-wise attendance percentage
- Attendance history
- Attendance trends
- Safe skip calculation
- Attendance goal planner

Examples:

Current attendance: 82%

If absent tomorrow:
80.3%

Safe to skip: Yes

Target attendance: 85%

Attend next 6 lectures continuously.

Attendance is tracked subject-wise and NOT globally.

Both WhatsApp and mobile app can update attendance.

======================================================================

5.3 Assignment Tracker

Purpose:

Simple reminder system.

Not a task management application.

Features:

- Create assignment reminders
- Due date reminders
- Pending assignments
- Completed assignments
- WhatsApp queries

Examples:

"DBMS assignment due Friday"

"How many assignments are pending?"

The system should only focus on reminding.

======================================================================

5.4 Notes Repository

Features:

Store:

- PDF
- PPT
- DOCX
- Images

User flow:

User forwards file to WhatsApp.

User mentions:

- Subject
- Unit

System stores automatically.

Retrieval methods:

WhatsApp:

"Send CN Unit 2 notes"

App:

Browse all notes visually.

Notes are organized:

Semester
↓
Subject
↓
Unit

======================================================================

5.5 Academic Calendar

Features:

Users can add:

- Holidays

- External exams
- Submission deadlines

Purpose:

7. REVIEW QUEUE SYSTEM

This is one of the most important features.

Purpose:

Human verification when confidence is low.

Examples:

"Spent ₹500"

AI knows:

Amount = 500

AI does not know:

Category

======================================================================

7.1 Rule 1

If only a SMALL piece of information is missing:

Ask immediately inside WhatsApp.

Example:

"Which account was used?

1. UPI

2. Cash

3. Savings"

======================================================================

7.2 Rule 2

If the user does not answer:

Send to Review Queue.

======================================================================

7.3 Rule 3

Complex ambiguities go directly to Review Queue.

Examples:

- OCR mistakes
- File classification uncertainty
- Multiple interpretations

======================================================================

7.4 Review Queue Location

Review Queue is APP ONLY.

Never inside WhatsApp.

12. AI PHILOSOPHY

AI MUST NOT be used everywhere.

Rule:

If deterministic logic can solve a problem, do NOT use AI.

Use:

- Arithmetic
- SQL
- Scheduling
- Business logic

before AI.

======================================================================

13. AREAS WHERE AI WILL NOT BE USED

- Attendance calculations
- Attendance prediction
- Attendance goals
- Timetable display
- Dashboard generation
- Finance calculations
- Monthly summaries
- Scheduled notifications
- Morning digests
- Night digests

These are deterministic systems.

======================================================================

14. AREAS WHERE AI WILL BE USED

14.1 Natural Language Understanding

Examples:

"Spent ₹250 on lunch"

"Present in DBMS"

"DBMS assignment due Friday"

"Send CN Unit 2 notes"

AI converts natural language into structured data.

======================================================================

14.2 Context Engine

AI prioritizes information.

Example:

Tomorrow:

3 lectures

2 assignments

Internal exam in 2 days

AI decides priorities.

======================================================================

14.3 Semester Drift Detection

AI detects long term patterns.

Examples:

- Falling attendance
- Rising expenses
- Delayed assignments

======================================================================

14.4 Cognitive Load Balancer

AI detects heavy weeks.

Example:

5 submissions

2 internals

AI suggests priorities.

======================================================================

14.5 Decision Assistant

Examples:

"Should I attend today's lecture?"

AI reasons using multiple data sources.
